How {{PartnerBrand}} runs a partner case

Six steps, all file-based. The partner stays in control. End-clients only ever see {{PartnerBrand}}.

1

Partner receives a catalog file

Inbound · file exchange

The end-client sends a catalog file to the partner using whatever channel the partner prefers (email, secure share, sftp). The partner forwards the file to {{PartnerBrand}} for scanning. No system access is granted on either side.

End-client does

Sends one catalog file in CSV, XLSX, or CWR format.

Partner does

Forwards the file to the {{PartnerBrand}} intake address.

{{PartnerBrand}} does

Acknowledges receipt within one business day.

2

Part 1 scan

Identifier · role · share · duplicate · conflict checks

Each row is checked for required identifiers, valid roles, share totals, duplicate works, and ownership conflicts. No corrections are applied at this stage — the scan is read-only.

3

Branded report and worksheet returned

Outbound · file exchange

The partner receives two files, both formatted under {{PartnerBrand}}: a Health Report (HTML) showing all findings, and a Worksheet (XLSX, CSV fallback) listing every row that needs a decision.

Partner does

Forwards both files to the end-client for review.

End-client does

Reviews the Health Report. Marks each row in the Worksheet: Accept, Edit, or Reject.

{{PartnerBrand}} does

Waits. No follow-up automation. The end-client decides if and when to return the Worksheet.

4

Partner returns the completed Worksheet

Inbound · file exchange

Once the end-client has confirmed each row, the partner forwards the completed Worksheet back to {{PartnerBrand}}. Only rows the end-client explicitly approved will be applied.

5

Part 2 corrections applied

Catalog rebuild from approved rows

{{PartnerBrand}} rebuilds a corrected version of the catalog using only the rows marked Accept or Edit. Rejected rows remain untouched. A second scan runs against the corrected catalog to confirm the fix.

6

Branded corrected report and exports

Final delivery

The partner receives four files, all under {{PartnerBrand}}: a second Health Report showing the residual issues, the corrected catalog, a CWR-format export template, and a CSV export template. The case closes.

Accepted file formats

{{PartnerBrand}} accepts the formats below for catalog intake. Partners may agree on a fixed format with each end-client.

FormatExtensionNotes
CSV.csvUTF-8, comma or semicolon separated. One row per writer.
Excel.xlsxFirst worksheet only. First row treated as header.
CWR.cwr · .v21 · .v22CWR v2.1 and v2.2 supported.
