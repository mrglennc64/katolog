# Kataloghub Validation Worksheet — XLSM Template Spec

The macro-enabled workbook that operators use to produce a correction
worksheet for HeyRoya. Output format mirrors HeyRoya's **Metadata Correction
Template** so the file round-trips between Kataloghub and HeyRoya without
schema translation.

## File

- **Format:** Excel Macro-Enabled Workbook (`.xlsm`)
- **Module:** `Module1` containing `CreateKataloghubWorksheet`
  (source: [scripts/kataloghub-worksheet.bas](../scripts/kataloghub-worksheet.bas))
- **Reference template:** `HeyRoya_Correction_Template.xlsx` (publisher-facing
  worksheet supplied by HeyRoya — defines the canonical column shape)
- **Contents:** structure only — no data

## Sheet

- **Name:** `Correction_Worksheet`
- **Initial state:** empty; structure is generated by running the macro
- The macro produces:
  - title block (rows 1–3)
  - instruction lines (rows 5–7)
  - table header (row 8)
  - data area (rows 9–508)
  - column widths, borders, freeze panes, sheet protection

## Title block (rows 1–3)

| Cell | Content | Style |
|------|---------|-------|
| A1   | `HeyRoya` | bold, 18pt |
| A2   | `Metadata Correction Template` | bold, 14pt |
| A3   | `For Music Publishers   ·   File-based correction workflow` | italic |

## Instructions (rows 5–7)

| Cell | Content |
|------|---------|
| A5   | `•  Publishers only edit the 'corrected_value' and 'notes' columns.` |
| A6   | `•  All other fields reflect the original metadata as received.` |
| A7   | `•  Return the completed file to HeyRoya for processing.` |

(Row 4 is intentionally blank as a visual separator.)

## Table columns (row 8)

| Col | Header | Source | Editable |
|-----|--------|--------|----------|
| A | `work id` | imported | no |
| B | `title` | imported | no |
| C | `writer name` | imported | no |
| D | `writer ipi` | imported | no |
| E | `role code` | imported | no |
| F | `share percentage` | imported | no |
| G | `publisher name` | imported | no |
| H | `publisher ipi` | imported | no |
| I | `STIM` | imported | no |
| J | `iswc` | imported | no |
| K | `isrc` | imported | no |
| L | `corrected value` | **publisher edit** | yes |
| M | `notes` | pre-populated by Kataloghub with validation issue; publisher may amend | yes |

For rows requiring multiple field corrections, the publisher writes them in
column L using `field=value` notation (e.g. `role_code=CA; share_percentage=100`).

## Data area (rows 9–508)

- 500 data rows, fixed range
- Locked metadata columns (A–K) shaded `RGB(242, 242, 242)`
- Editable columns (L, M) on white background
- All cells gridlined with continuous borders

## Column widths

Matches HeyRoya's reference template:

| Col | Width | Col | Width |
|-----|-------|-----|-------|
| A   | 14 | H   | 14 |
| B   | 28 | I   | 14 |
| C   | 22 | J   | 18 |
| D   | 14 | K   | 18 |
| E   | 12 | L   | 28 |
| F   | 19 | M   | 32 |
| G   | 24 |     |    |

## Protection

- All cells locked
- `L9:M508` unlocked (publisher edits corrected_value and notes)
- Sheet password: `kataloghub`
- Filtering allowed

## Operator workflow

1. Open the master `.xlsm` template.
2. Run `CreateKataloghubWorksheet` (button or `Alt+F8`).
3. Paste imported catalog rows into `A9:K508` (the unlocked import range on a
   freshly built sheet).
4. Pre-populate column M (`notes`) with Kataloghub validation issues for any
   row that has a problem.
5. Save a working copy as `.xlsx` and deliver to HeyRoya / publisher.

## Distribution

- The master `.xlsm` template lives with the operator team.
- Returned worksheets are saved as `.xlsx` (no macros) — only the master
  template remains macro-enabled.

## CSV equivalent

The same 13-column shape is used in both directions of the round-trip:

| Stage | File | Purpose |
|-------|------|---------|
| Out — empty | [samples/validation-output-template.csv](../samples/validation-output-template.csv) | Header + comment block, no data |
| Out — sample | [samples/validation-output-sample.csv](../samples/validation-output-sample.csv) | Kataloghub-generated worksheet with issues flagged in `notes`, `corrected value` blank |
| In — empty | [samples/correction-input-template.csv](../samples/correction-input-template.csv) | Header + comment block describing the returned-file conventions |
| In — sample | [samples/correction-input-sample.csv](../samples/correction-input-sample.csv) | Same shape with `corrected value` populated on edited rows |

The CSV uses a leading `'` on numeric identifiers (`writer_ipi`,
`publisher_ipi`) so Excel preserves them as text rather than coercing to
scientific notation.

For rows requiring multiple field corrections, the publisher writes them in
`corrected value` using `field=value` notation, semicolon-separated
(e.g. `role_code=CA; share_percentage=100`).
